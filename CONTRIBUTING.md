## How to Contribute

Thank you for your interest in contributing to AHS Agentic! This project is proprietary software, which means all contributions must be properly authorized.

### Before Contributing

1. **Contact the Project Owner**: Reach out to discuss your proposed contribution
2. **Sign a Contributor License Agreement (CLA)**: This ensures proper legal protection
3. **Get Explicit Authorization**: All contributions must be explicitly authorized

### Contribution Process

1. **Fork the Repository** (if authorized)
2. **Create a Feature Branch**: `git checkout -b feature/your-feature`
3. **Make Your Changes**: Follow the existing code style and conventions
4. **Test Your Changes**: Ensure all tests pass
5. **Submit a Pull Request**: Provide a clear description of your changes

### Code Standards

- Follow PEP 8 style guidelines for Python code
- Write clear, concise commit messages
- Include tests for new features
- Update documentation as needed
